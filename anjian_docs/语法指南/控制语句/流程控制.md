## 线程控制语句

### EndScript - 停止当前线程

`EndScript` 命令用于强制结束当前线程的运行。其作用范围取决于执行环境：

*   **主线程中执行**：停止整个脚本的运行
*   **子线程中执行**：仅停止当前子线程，主线程和其他子线程不受影响

#### 停止主线程示例

```vbscript
TracePrint "脚本开始执行"
EndScript
TracePrint "上面强制停止了，这里不会输出"
```

#### 停止子线程示例

```vbscript
Function ThreadFunc()
    Dim idx = 1
    Do
        If idx >= 5 Then 
            EndScript  // 只停止当前子线程
        End If
        TracePrint "子线程：" & idx
        idx = idx + 1
        Delay 1000
    Loop
End Function

Thread.Start ThreadFunc
Do
    TracePrint "主线程运行中……"
    Delay 1000
Loop
```

在上面的示例中，当子线程中的 `idx` 达到5时，只有子线程会停止，主线程继续运行。

### ExitScript - 停止整个脚本

`ExitScript` 命令会强制停止所有线程的运行，包括主线程和所有子线程。

#### 使用示例

```vbscript
Function ThreadFunc()
    Dim idx = 1
    Do
        If idx >= 5 Then 
            ExitScript  // 停止所有线程
        End If
        TracePrint "子线程：" & idx
        idx = idx + 1
        Delay 1000
    Loop
End Function

Thread.Start ThreadFunc
Do
    TracePrint "主线程运行中……"
    Delay 1000
Loop
```

在这个示例中，当子线程中的 `idx` 达到5时，整个脚本（包括主线程和子线程）都会停止运行。

### EndScript vs ExitScript 对比

| 命令 | 作用范围 | 使用场景 |
| --- | --- | --- |
| `EndScript` | 仅停止当前线程 | 需要停止特定线程但保持其他线程运行 |
| `ExitScript` | 停止所有线程 | 需要完全终止程序的所有执行 |

## 脚本重启控制

### RestartScript - 重启脚本

`RestartScript` 命令会立即停止当前脚本的执行，并从头开始重新运行整个脚本。在多线程和悬浮窗事件中也可生效

#### 使用示例

```vbscript
TracePrint "脚本开始执行"
Delay 3000
// 重新启动脚本
RestartScript
// 以下代码不会被执行
TracePrint "这行代码永远不会被执行！"
```

#### 注意事项

*   `RestartScript` 执行后，其后的所有代码都不会被执行
*   脚本重启后，所有变量和状态都会重置
*   适用于需要重新初始化程序状态的场景

## 代码跳转语句

### Goto - 无条件跳转

`Goto` 语句实现无条件跳转功能，需要配合 `Rem` 标记使用。执行 `Goto` 后，程序会直接跳转到指定的标记位置继续执行。

#### 基本语法

```vbscript
Goto 标记名
// 这里的代码会被跳过
Rem 标记名
// 程序会直接跳转到这里继续执行
```

#### 使用示例

```vbscript
TracePrint "程序开始"
Goto 跳转点
TracePrint "这行不会被执行"
Delay 10000  // 这个延迟也会被跳过

Rem 跳转点
TracePrint "直接跳到这里了！上面的延迟没有被执行！"
```

### Rem - 代码标记

`Rem` 用于在代码中设置标记点，作为 `Goto` 语句的跳转目标。

#### 标记命名规则

`Rem` 标记的命名必须遵循**标识符**命名规则：

*   由字母、数字、下划线组成
*   不能以数字开头
*   不能使用系统保留字
*   建议使用有意义的名称

#### 标记示例

```vbscript
Rem 开始处理
// 处理代码

Rem 错误处理
// 错误处理代码

Rem 程序结束
// 结束处理代码
```

## 流程控制最佳实践

### 1\. 合理使用线程控制

```vbscript
// 推荐：在子线程中使用EndScript
Function WorkerThread()
    // 工作逻辑
    If 任务完成 Then
        EndScript  // 只停止当前工作线程
    End If
End Function

// 在主线程中使用ExitScript终止整个程序
If 发生致命错误 Then
    ExitScript  // 停止所有线程
End If
```

### 2\. 谨慎使用Goto语句

```vbscript
// 不推荐：过度使用Goto导致代码混乱
Goto 标记1
// 大量代码
Rem 标记1
Goto 标记2
// 更多代码
Rem 标记2

// 推荐：使用函数和循环结构
Function ProcessData()
    // 清晰的函数逻辑
End Function

For i = 1 To 10
    // 使用循环替代重复的Goto
Next
```

### 3\. 错误处理中的跳转

```vbscript
// 合理使用Goto进行错误处理
If 检查条件失败 Then
    Goto 错误处理
End If

// 正常处理逻辑
TracePrint "正常执行"
Goto 程序结束

Rem 错误处理
TracePrint "处理错误"
// 错误处理代码

Rem 程序结束
TracePrint "程序结束"
```

## 总结

*   **EndScript**：用于停止当前线程，适合多线程环境下的局部控制
*   **ExitScript**：用于停止整个脚本，适合程序的完全终止
*   **RestartScript**：用于重启脚本，适合需要重新初始化的场景
*   **Goto/Rem**：用于代码跳转，应谨慎使用，优先考虑结构化编程

选择合适的流程控制语句可以让程序逻辑更清晰，执行更高效。
